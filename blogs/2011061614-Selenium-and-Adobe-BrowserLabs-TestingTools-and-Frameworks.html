Selenium and Adobe BrowserLabs - TestingTools & Frameworks

Megha Agarwala

June 16, 2011

I have been experimenting with Selenium to build unit test cases for Ednode and also been looking at AdobeBrowserLab which helps you to check for rendering of your web pages in different browsers and Operating Systems. Selenium is a free software testing framework for Web Applications. It allows you to record/write tests in a number of different programming languages and play them from many different Web browsers. There are primarily three components to Selenium - 1. Selenium IDE : An integrated development environment, which comes as a firefox extension. it allows you to record, edit and debug your test cases. Scripts may be automatically recorded and edited manually. It also provides autocompletion support. 2. Selenium Remote Control (RC): Runs your tests in multiple browsers & multiple platforms. 3. Selenium Grid: distributes your tests across multiple servers, minimizing the test execution time. Adobe Browser Labs Generates screenshots of browser renderings in real time, with multiple viewing options to check for cross compatibility issues So far, I feel that Selenium and Adobe BrowserLab can tremendously reduce our testing effort and help the team to check-in code more often and fearlessly. Does anyone have any experience with Selenium/BrowserLab or any other automated testing tools?
