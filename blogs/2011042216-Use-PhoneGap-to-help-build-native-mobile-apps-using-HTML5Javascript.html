Use PhoneGap to help build "native" mobile apps using HTML5/Javascript

fanwar

April 22, 2011

We've been talking a lot about moving to HTML5 for Vialogues and eventually having more robust support for vialogging using mobile devices. One challenge we've often run in to is making a choice about a web application (run inside of the mobile browser) vs. native applications. Native apps are inherently more powerful and have more features available, but web apps are much quicker to develop and can be ported across multiple environments. PhoneGap seems like a promising framework to help get the best of both worlds. It lets you build native apps, but using html/html5, javascript and other standard web technologies. Looks like a great way to rapidly prototype new mobile apps, even if we decide to go with native apps on some popular platforms eventually.
