OS is a resource manager providing services using system calls. CPU mode has two modes 1) User Mode 2) Kernel Mode Some of the system calls are for 1) Processes - fork() fork creates a process and returns after the fork completes ( main() { pid = fork(); if (pid == 0) { fork(); } while(1); } creates three processes instead of 4 as only the child process creates. unistd.h read, write ->close ex. open("/usr/foo/dus.txt",O_RDONLY) returns file descriptor is a handle returned by a file. a file descriptor is just an integer and every time you open it's just an integer(refer a way which file). Can a program be going to two cpu's for instruction. like a webserver. Interrupts OS checks for interrupts between execution of the scheduled processes. Interrupts could be from Discs, Keyboard, Network - There is a interrupt handlers. These interrupt handlers are nothing but operating system code and causes the switch from user mode to kernel mode and then switch it back from kernel mode to user mode. Interrupts are not buffered but devices have their own buffered. How to differentiate between different lines as the CPU knows there is an interrupt but not what type of it? It would serially check for all the interrupts. why is there a lag sometimes between a keyboard press and the response from the OS? two modes of protection 1) CPU mode - kernel mode and user mode. 2) Timer interrupts - gives the OS a chance to gain back the control of the system. Interrupt handler are written in assembly language, so that there is maximum performance gained by leveraging the hardware. There is a priority associated with the Interrupts. disabling interrupts is called masking. - If you are in the middle of an interrupt and there is an interrupt which comes up you want to disable the interrupt. - the cpu stops checking if there is a signal happened for an interrupt. - we can disable certain types of interrupts. and enabling interrupts is called unmasking
