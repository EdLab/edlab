Citation Parser for LIVE syllabus

Huning Dai

April 17, 2009

In LIVE syllabus, one of the major features is to help the lecturer upload their reading list easier than the way provided by Docdel. Now by using Docdel, the lecturer has to manually input each fields (Title, Author, Year, Pages, etc.) of the reading. While the LIVE syllabus' approach is to have the lecturer upload a MS word document, then the system will analyze the uploaded file, take out each field and filled the form automatically. Earlier this week, I manage to write a prototype website to implement this kind of idea. However, it requires strict formatting of the citation that you wrote in the word document. Feel free to try it. Notice that the system can only support one citation at one time. As the system should accept all types of format of the citation and should be able to separate each fields of the citation, I did some research on “reference parser” and “citation parser” on the internet. I was very surprised when I realized this job isn't easy as I thought it was going to be fairly easy. Many colleges and universities started projects on related topics since 1998, and they had made lots of progress in this. There are two main kinds of ideas to achieve citation parsing. One is using an external database storing the information of all the papers, journals and books. Here by saying information actually I'm not including the content of the citation, but rather the title, authors, the year and other kinds of general information. So it won't require a huge database, however it won't be small neither. The citation uses the information the user provides to generate a query in the database. Sounds really intuitive and highly effective, right? It also has a good performance in accuracy. I believe that Google is using now for citation counting in Google scholar. The most successful production of this idea in academia is CiteseerX. The other one uses a technology in information processing called Sequence Alignment Technique. Due to my understanding of this technology, it uses the special pattern of the citation to recognize each field of it. This idea will not require external database, therefore by sacrificing quite a little accuracy (according to the paper, the test result is only 1-2% lower than citeseerX), the system becomes simpler. After reading about all this, I am trying to find a way to use both kinds of ideas in LIVE syllabus, as I am now working on producing an intelligent parser for DocDel. If anyone has any great ideas to share with me, it will be really appreciated.
