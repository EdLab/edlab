Let's Make NLT Better

Yudan Li

September 20, 2012

This was originally going to be a reply to Zhou Zhou's post about W3C testing of NLT. But I'm doing a separate posting here as I thought it could actually be a good place to mention some of our findings in NLT mobile developing and our approaches to address existing issues. Thanks Zhou Zhou for finding W3C tool for mobile app performance testing. From the results, we see there's a big room for NLT mobile to be improved. NLT is our first attempt in developing HTML5 mobile apps. Many parts of it were designed and developed using our existing web developing experience. Indeed, we experimented and adopted many new mobile-specific techniques but there are a lot more we need to use. According to W3C's testing results for other popular sites, it can be seen W3C considers a good mobile app should be developed using a very different set of standards and tool set compared to web apps. NLT uses one single backend to serve its two frontends. One possible optimization approach we can take is to use a more mobile-oriented backend for the mobile frontend. That will typically reduces SOAP connections and increases REST connections and thus increasing loading speed. The widely use of AJAX on the frontend also requires the backend to be able to serve light-weighted response for multiple purposes, which is featured by REST framework. For the UI developing, we have been adopting a more mobile-specific approach. We have been boldly using JQuery Mobile framework. It proved successful in emulating some of the mobile page transition effects. And it's also been successful in handling some mobile-exclusive events, like rotating and swiping detecting. However, we do face 2 major issues on the UI part. The first is efficiency. We have seen lags on some of the pages. This happens mostly in page loading and page swiping. We're going to address this issue by optimizing and simplifying the existing code. We are going to reorganize some of the existing objects and functions to make sure less redundant information is loaded in page accesses. To further enhance the performance, we're going to use more pre-fetching and caching techniques, which will speed up page loading. The second major issue is compatibility. NLT app doesn't work as good on Android phones as on iPhone. This is partially because most of the UI developing was done against iPhone and iPhone simulator. Our previous assumption was that significant difference might only happen when a different browser was used. However, it turns out mobile Safari browsers behave very differently between iPhone and Android. In future, we are going to do better testing by using Android or Android Emulators. Thank all again for finding bugs and giving suggestions during yesterday's seminar. We're going to fix all the bugs asap and carry out more new features.
