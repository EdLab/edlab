Chrome Frame - A Viable Alternative for Avoiding Development on the IE?

Alex Park

July 15, 2011

Developers! Do you hate developing for IE? Well, I hate it. It's so inconsistent and standard-INcompliant (thesaurus.com tells me that's not a word) that you have to write an entire set of CSS rules just to make it work on IE. Also, since the world is not perfect, developers make mistakes. However, when those mistakes are made, IE is far less forgiving, usually choosing to throw out a bunch of code from display rather than attempting to understand what the developers meant. The other browsers are a lot more forgiving, however, so these problems usually need to be debugged only on the IE. To make things worse, IE 8 and below has nothing that resembles Firebug or Inspector in terms of functionality, which makes debugging that much harder. (continued) So why do developers support it? Well, it's because it is still the most widely used browser in the world. According to Wikipedia, IE still has more than 40% market share in the world. (I'm also ashamed to report that a scary amount of people in Korea still use IE 6, which has been touted as the devil multiple times) This means developers have to continue developing for IE for the foreseeable future. The good news is, IE 9 made huge strides in following standards, so developing for IE 9 is much easier than developing for any other IEs of the past. So why don't we just develop for IE 9 only? Because Microsoft made IE 9 available only for its Windows 7 users. So, I was excited when I heard about Chrome Frame. It came out in the days of (now discontinued) Google Wave, when it wouldn't work in IE 6. Basically, it injects Chrome's rendering and javascript engine into IE, making IE act like, well, Chrome! It would sure make developing for IE a lot easier, well, because you dont' have to anymore. Of course, all good things in this world have a flip side. Since this is a plugin, some experts (most notably Microsoft) have claimed that this makes the browser less secure (although this is not the biggest of your concerns if you are using IE 6). Also, the user has to download a plugin for Chrome Frame to work, which may or may not be annoying. What are your thoughts? Would this be a good way for developers to stop developing for IE without dropping "support" for it?
