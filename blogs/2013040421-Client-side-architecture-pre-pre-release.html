Client side architecture: pre pre release

Sharath Gururaj

April 4, 2013

private please Hi Team, I have a version of the new client side architecture. Since it involves some new technologies, This post is meant to list out the technologies involved so that developers can read up on it (if you arent already familiar) and help improve/give feedback on the architecture. The architecture is designed to meet the requirements of this post The key technologies used are:

  • RequireJS for loading javascript (and even html, css) files as required, which allows us to split up code into multiple files, specify dependencies between files, and load files in the browser lazily. Consider it the import statement of javascript.
  • Angular JS While I must admit that angular presents a rather steep learning curve, It offers payoffs if we do things the right way. We can start off slow by adopting key features like angular modules and then incorporate other features.
  • Jasmine The tests will be written in jasmine. Its lightweight and has good support. Also, the test cases read like a story so its easy to tell what test case is failing.
  • Karma A test runner. Basically, watches for changes for files and reruns test cases in browsers, leading to instantaneous feedback. We're talking milliseconds, not seconds or minutes. The experience is quite amazing.
I recommend brushing up your javascript knowledge by reading any of the good javascript books available, it is simple, but in its simplicity lies a lot of power and opportunities for subtle bugs. Briefly, The architecture uses a stub html to kick things off. There is a main entry point js file which uses requirejs and angular modules and routes to load a particular view. The code asynchronously fetches html template, fills in the data and renders it to the view. In the coming days, we can start rolling out the framework and work together to improve it further.
