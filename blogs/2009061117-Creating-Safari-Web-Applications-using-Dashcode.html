Sarah Odwalla Dashcode Engineer - Dashcode 2.0 (used for creating web applications for iPhone) -Dashcode 3.0 has support for : # Safari Web Applications can be created in #podcast, rss , etc templates to get started # Webkit inspector can be used for inspection HTML 5 - Application cache - offline web application (the browser knows what to render) video src="getamac.mp4" Dashcode has support for :- 1) Reflections 2) Transitions 3) Animations 4) Transforms New UI parts: 1) Split Layout 2) Grid Layout - good for collection of items like movie 3) List - Keyboard navigation, multiple and empty list selection Bindings- where the XML data is directly taken in and used for reflection. Dashcode Bindings of Data Sources using Dashcode Core Functionality - Key Value observing - Bindings - Central event handling - Touch Support DC.KVO - is the main class. JavaScript class Basics -does not include support for class based objects but dashcode does that. Data Sources - Instance of DC.AjaxController - Fetches on load Changing Data Source Parameters var dataSource=dashcode.getDataSource('trailers'); dataSource.setValueForKeyPath('G','parameters.raing'); Getting Data -------------- dataSource.valueForKeyPath ('content.info.actors'); State: ----- dataSource.addObserverForKeyPath({},function(change)){ console.log('Error fetching trailers/'+change.newValue);},'errorMessage'); Collection Data Sources: 1) Instance of DC.ArrayController - Sorting - Selection Management - Filtering 2) Accessed via DC.getDataSource Bindings ----------- Value Transformers are used to transform the data when the stored data is not in the same format as the output data should be. Setting Value Transformers - can be done using Value Transformer and there are several built-in which can be used. A new valueTransformer can also be created. Placeholders are used for selection bindings- what happens if the user selects two list ids what data should be displayed to the user (this can be don in the bindings tab named Placeholders).
