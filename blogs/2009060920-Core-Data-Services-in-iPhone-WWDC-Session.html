Core Data Services in iPhone (WWDC Session)

pramnab1

June 9, 2009

Introduction to Core Data on iPhone --------------------------- Core Data is a powerful and efficient framework. Problemset: Backend of the application to be managed with the speed and efficient - to make the data persisted Introduction to Core Data Technology 1) Modeling your data 2) Working with objects 3) Displaying results Overview of Framework: 1) Efficient fetching and saving 2) Changing tracking and undo 3) Validate the objects 4) Performance Model is the first requirement Store = Data File (Actual Objects (Objective C objects are called as managed objects)) Managed Object Context - higher object stacks The Core Data Workflow: Introducing "The Stack". NSManagedObjectModel ->NSPersistentStoreCoordinator requires two things 1) NSPersistentStore 2) Store File Once we do this we create an object NSManagedObjectContext with a connection with database using NSFetchRequest. NSManagedObjectModel - ER MODEL NSFetchRequest - Select From Clause (SQL QUERY) Getting a working stack 1) Load Model 2) Create Coordinator and set model 3) add raw data 4) Create NSFetchRequest
