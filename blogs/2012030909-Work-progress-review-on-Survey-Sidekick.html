Work progress review on Survey Sidekick

kiraluo163

March 9, 2012

There are several progress on the development of Survey SideKick I would love to share. 1. I finished the backend model, and implemented new frontend design for "Launch" page of Survey Sidekick, which enable users to choose a date to launch the survey and also close the survey. I ported over the built-in Calendar module, and integrated it on Python and Django platforms. I also finished the backend model for the user to send a custom invitation and reminder messages. 2. I also implemented the new UI design for sample page, and finished the backend model of creating and editing the email list, and adding or deleting the individual email address of the recipients of that survey. I created a pop-up curtain for user to view and edit the email address list binded with that survey. Also finished the API so I can now proceed with the ajax request for "onfailure" and "onSuccess" case. The email and name validation process that is created will validate user's email and name input, and stored in database after it pass, and return the error message if it is not in a right format. 3. I also finished the implementation of CAS user login system for Survey Sidekick. I ported over the module from previous projects like Vialogues and make it function on Survey Sidekick. Right now users can login with their CAS accounts. 3. I got hands on designing and implementing the backend model for three new question types: likert list question, question with embedded picture or other media and likert question combined with free-response text. So far I finished designing and creating the database model for the first new question type and create a group of APIs of backend for users to create the likert list for each likert question.
