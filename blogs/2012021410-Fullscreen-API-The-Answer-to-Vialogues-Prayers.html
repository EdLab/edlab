Fullscreen API... The Answer to Vialogues' Prayers?

Alex Park

February 14, 2012

The advancement of Web standards (and further decline of Internet Explorer model of releasing a version every decade <= this is an exaggeration) can be said to have been caused by Google Chrome's aggressive auto-updating, fast-releasing strategy. In fact, Firefox has also jumped on the rapid release bandwagon; in fact, it took Firefox a full seven years to go from version 1.0 to 4.0, but upon being influenced by Chrome to change their release cycle, they have hit version 10.0 in less than a year. Of course, we should note that there are far few "major" changes version to version as a result. However, no one can deny the advantage of doing this, which is that new features can be pushed out much faster. One of these features is the Fullscreen API. Once just a cool idea, now half of the browsers support it. And it's something that I think can help Vialogues immensely. We've always had problems with Vialogues and smaller screens. The fixed left pane causes the manage, embed, and tweet buttons to be inaccesible when the browser window is small. With the Fullscreen API, however, we could give users the option of using Vialogues with the entire screen real estate, which should more or less eliminate this problem for small-screen users.
