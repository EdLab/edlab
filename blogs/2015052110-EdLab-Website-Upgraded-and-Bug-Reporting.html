EdLab Website Upgraded and Bug Reporting

zhou

May 21, 2015

Hello EdLabbers and Friends, Welcome to the upgraded EdLab website! Thanks to the support from Hui Soo and D&R colleagues, we finally made it migrating the EdLab website from TC to AWS (in April), upgrading the Drupal CMS from v.4.6.4 to 6.35, and moving from the Windows server to Linux server! To me the biggest challenge was making the new website look as close as possible to the old one given the barriers of: 1) The old website altered the source code of Drupal to achieve some functionalities, which is not honored by Drupal development protocols and has to be revamped in the standard way; 2) A few plugins used in the old website which had played a core role (e.g. Events, Publications, Projects) are discontinued by the developer. So I have to find alternative ways to enable these node types and migrate the data over. Due to the factors above, there are things different in the new website which I cannot avoid. And there are intentional changes too for good. Below is a summary of the major differences (appearance & functionality) in the new website: 1) The new webpage frame is wider so as to look better in modern computer screens, and to avoid embedded Vialogues from overlapping the sidebar. 2) Improved/slightly different fonts and styles. 3) Some different options for publishing, especially for the administrator accounts. E.g. a) Javascript date picker for event times (in comparison to text inputs). b) Projects have a new “thumbnail” field. A circled picture should be uploaded as the thumbnail of the project. 4) Adopted localized timezone settings such as America/New_York instead of the ETC timezones. The difference is the former has daylight saving built in, while the latter doesn’t. So from now on we don’t have to switch the timezone from ETC -4 to -5 (but to be honest we never seem to care about this) while creating future events. 5) Finally on the technical side: a) Drupal v.4.6.4 upgraded to 6.35. We are not upgraded to the latest Drupal version at this time because more plugins are discontinued in Drupal 7 and 8. We have to perform further upgrading at a later step. b) Server is migrated from Windows Server 2008 to Ubuntu 14 with Nginx (more secure, more stable) and RDS MySQL database. c) The theme codes are revamped so that the Drupal core is no longer hacked. This way it’ll be very easy for us to upgrade to higher Drupal versions. d) Re-written theme stylesheets using LESS. Future works: 1) A WYSIWYG editor. Currently Markdown is being seriously considered. Please refer to my earlier post and provide your vote. 2) Upgrade Drupal to the latest version. 3) Improve the theme codes and use mobile-friendly responsive webframe. I am sure there are bugs after the upgrade. Please reply to this post with any bug you find and I’ll fix it. Thank you! ----------------------------------------------------------------- P.S. While publishing a post in the future, if your account allows you to choose "Publishing options", please do not check "Sticky at top of lists" unless you feel absolutely sure. If that option is checked, your post will always stay on top of newer posts. Thank you.
