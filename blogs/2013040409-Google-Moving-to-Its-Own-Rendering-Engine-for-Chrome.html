Google Moving to Its Own Rendering Engine for Chrome

Nara Kasbergen

April 4, 2013

The biggest bane of the web developer's existence is browser inconsistencies. It's painful to spend hours working on something, then test it in a different browser and find out that the CSS is completely off, or that the JavaScript code won't run. The situation has gotten slightly better in recent years, as more and more browsers have become increasingly standards-compliant, and one of the nice things has been that at least Chrome and Safari both use the same rendering engine, WebKit, so if it works in Chrome, you can assume it also works in Safari. Well, unfortunately, soon that will be no more. Google has announced that it plans to fork WebKit and develop it into its own rendering engine called Blink, which will soon power the Chrome browser and ChromeOS. I'm sure Google's reasons for the decision are legitimate and we're likely to see significant improvements in Chrome over time as a result, but overall I still find the news unfortunate. No longer being able to blanket-assume that if it works in Chrome it works in Safari as well will make developers' jobs just a little bit harder.
