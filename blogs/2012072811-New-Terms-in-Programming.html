New Terms in Programming

Josh Lopez

July 28, 2012

(Keep this private, if for no other reason that because no one outside EdLab is going to get it.) In a recent post on his blog Coding Horror Jeff Atwood, one of the creators of Stack Overflow, shows the top 30 "new" programming terms that people have created and popularized around their offices. The descriptions are pretty funny, and there are a few dos and don'ts included that I think a lot of us can be proud/guilty of. Here's a few I thought were particularly applicable: 17. Hydra Code - Pocket Knowledge 21. Smurf Naming Convention - If Data Dashboard is ever renamed, it's going to have to be to something with D.D. initials. 23. Rubber Ducking - Pranav is good a letting you do this. 28. Megamoth - I've spotted a few of these flying around Vialogues. Have you seen any of these, or any of the other terms, coming to life around TC?
