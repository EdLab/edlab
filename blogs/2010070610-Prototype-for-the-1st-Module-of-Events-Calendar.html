Prototype for the 1st Module of Events Calendar

Haomin Zhu

July 6, 2010

As Pranav proposed before (One Events Calendar for TC Community), I developed a prototype for the 1st Module of Events Calendar.

Basically, the Events Calendar project could be separated into 2 modules. The 1st module called aggregator takes charge of gathering events data from multiple events calendar. The 2nd module will provides personalized calendar service, such as adding or subscribing events in personal calendar, post events to personal calendar, invent others to an event and so on.

Currently the prototype is only for the 1st module, aggregator. In the prototype, I use Teachers College's, Barnard's and School of the Art's calendar as sample data sources. The first two calendars could be subscribed by using RSS feeds, while the last one is pure HTML which cannot be subscribed. Thus, when using the aggregator to view the events, we do not need to go to every calendar to know the coming events, but just stay at our Events Calendar.

In the 1st module, aggregator, we will finish the following features:

  • user views all events in real-time from multiple data source
  • user views an event in detail
  • user filters events by using different kinds of filters
  • user searches events by keywords
  • user shares events by using email or social networking
