Data Dashboard is moving forward!

Tian Xie

April 26, 2010

We are making progress with Data Dashboard! After implementing the Library Website on the Dashboard, we are now working on the Library Patron Traffic Report. The situation of the Library Patron Traffic Report is a little special. Unlike other applications, we are using a little device called "Count Manager" to collect Patron Traffic data from seven sensors in the library. The data is stored in text files in the "Count Manager", and a laptop is connected with it so that we can get data downloaded onto the laptop. However, the laptop is not connected with any servers and so the report writers have to manually collect the Patron Traffic data. The first step we took to resolving this issue was writing a script on the laptop and setting up a task scheduler for the script to run once every month, which automatically then downloads the data file from "Count Manager" at the given time in the month. In the mean time, a database has been created to store the Patron Traffic data in one of our database servers, which makes it much easier to manipulate the data. The Dashboard now is able to communicate with the database that stores the Patron Traffic data. The next step is connecting the laptop with the database server. And with the help of another script, we can have the Patron Traffic data automatically inserted into the database. We are moving forward.
