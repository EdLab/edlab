Search Done Right

Dan Mallinger

January 10, 2007

So what have I been doing this week? Well, a bit of everything but this last item is really cool: Check out the PK Development site (on campus only) and take a look around. Just a few files: Gary's CV, a PDF book on fun things you can do with Google, a paper on "summer slide," a great song (actually by the Old 97's, not Rhett Miller solo), a paper on the EM algorithm (remember that really complicated posting I made a while back on EM? it returns!), and a couple others. The files are labeled horribly. No tags, titles consist of the word "file" and not much more, no communities, etc. So how in the world could I find Gary's CV? Oh the horror!! But wait... what if I searched here... It's a Christmas miracle! (UPDATE: currently this miracle only accessible from WIRED machines in the EdLab - no wireless access.) Now try searching for: Google Expectation Gary And maybe try education... This new search box captures the content of PK as well as parses, indexes, and (optionally) caches stored files of many types including PDF and doc. It's like Google, only better because we control the appearance of the search. With a little work, it can be completely integrated into a site so that it looks just like, for example, the TCRecord search. The only caveat is that we're limited to 500,000 pages per server. I'd like to start a discussion about our search needs and where this new tool could fit in. It's not just about fast search or indexing PDF's and .doc files. Consider the time spent when we make new sites to have some search functionality (and isn't it still missing from the library site?). Setting up a clean system, a reusable code library and some administrative processes could save a lot of time in the long run and make our sites look more professional.
