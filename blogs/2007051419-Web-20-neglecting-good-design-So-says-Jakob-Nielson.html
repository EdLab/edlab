Web 2.0 'neglecting good design' - So says Jakob Nielson

Stephen Asunka

May 14, 2007

Jakob Nielson, (described in various circles as a web usability guru), is arguing that the hype about Web 2.0 technology is resulting in poor website design. In an article posted on bbc.co.uk, Nielson points out that, in designing contemporary (or Web 2.0 compatible) websites, web firms are focusing more on the personalization tools of the Web 2.0 era (i.e. user generated content, folksonomies, personalized/dynamic webpages, virtual communities etc), whilst neglecting the basic practices of good website design (i.e. easy search and navigation, clear fonts, appropriate graphics and colors etc). Nielson's position however is that, since research into website use has shown that sites are better off when they get the basics right, Web 2.0 technologies should be made secondary to the basic issues. Nielson further supports his argument by alluding to the 1% rule (a rule that proclaims that for any group of users of a website, only 1% contribute content, whilst almost 90% never contribute anything), and implying that focusing on Web 2.0 technologies will mean serving a small committed minority, whilst making a site far less useful to a vast majority of users. Though good design and Web 2.0 compatibility are not mutually exclusive outcomes of a website, I realize that we may still need to keep Nielson's argument "at the back of our minds" when we embark on creating web-based resources for communal use.
