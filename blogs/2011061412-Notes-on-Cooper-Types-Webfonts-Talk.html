Notes on Cooper Type's Webfonts Talk

Amelia Grohman

June 14, 2011

I attended last night's talk on webfonts hosted by Cooper Union's Type Design program, and wanted to share some of my notes with anyone who may be interested in this subject. Tim Brown from Typekit and Nick Sherman from Webtype represented their webfont hosting and distribution services. Scott Kellum represented Treesaver, which is an HTML5 in-browser app designed for publishers to create typographically sophisticated reading experiences for multiple platforms and devices simultaneously. All of their sites are great starting points for learning about webfonts and improving readability and design on the web. They discussed the WOFF font packaging format, which is a compression wrapper that adds XML metadata to a font file. This enables more portability of webfonts while adding protection from unauthorized use. There was also much talk about hinting, which is the obscure and expensive art of determining how vectors translate into pixels in Windows systems, and notorious for making type appear awful if done poorly. Webtype's RE typefaces and Microsoft's latest wave of screen fonts were designed to be readable on Windows and can be studied for comparison. There are a number of applications such as Adobe Browser lab that can be used to see this in action. Both Typekit and Webtype offer 30 day free trial periods to test fonts before purchase, and many free (some of dubious quality) fonts are available through Google Fonts. The takeaway seems to be that there will be a great proliferation of webfonts available, but that building a truly useful, robust typeface will, for the time being, continue to be a slow, expensive undertaking and labor of love by extremely skilled professionals whose critical work should be valued and supported. A few more resources for finessing webfonts in action: lettering.js and kern.js are jQuery plugins that enable designers to tweak spacing between pairs of letters as they would in a desktop environment. Webfont developers are still working on enabling OpenType features and individual instances of contextual alternates, which are difficult problems to solve. Typekit's font events, aka Google's Webfont Loader are JavaScript libraries intended to allow more control over transitional states when using Webfonts. There has been improvement lately, but you may be familiar with the Flash of Unstyled Text (FOUT) phenomenon, in which type reverts to a web default face until it finishes loading (often in FF). This is especially problematic when the design calls for condensed typefaces, which have no spatial equivalent in web defaults. Lest the print designers of the world get overzealous and try to destroy the web with their desire for control, the major takeaway from the event is that all web presentation is optional and separate from content. It is critical that design be effective, beautiful and a great enhancement to the user's experience, or it risks being stripped away. This moment in history is a perfect opportunity for web developers to fall in love with typography, and for typographers to fall in love with web technology.
