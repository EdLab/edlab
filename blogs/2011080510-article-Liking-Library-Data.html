article: "Liking Library Data"

Julia Martin

August 5, 2011

Another one for the gossip books, but this is librarian gossip at the very least. So the author of this article thinks we should be embedding library data (catalog/holding data specifically) into social networking sites due to their contributions to the Semantic Web and the automatic inclusion in Search Engine results. then the author says this: "Now here's a big problem. Once you put the like button javascript on a web page, Facebook can track all the users that visit that page. This goes against the traditional privacy expectations that users have of libraries. In some jurisdictions, it may even be against the law for a public library to allow a third party to track users in this way. I expect it shouldn't be hard to modify the implementation so that the script is executed only if the user clicks the "Like" button, but I've not been able to find a case anyone has done this" As someone indoctrinated into the Society Of Library and Information Science, I have to say, this is a bigger issue than he's casually mentioning. Public libraries won't even give user data to the US Government (and for good reason). Particularly with Facebook selling user data, and google using similar data to adjust and anticipate user's search result needs and behaviors potentially causing a search tunnel. On the other hand, go to the users, go to the patrons, for every book his reader, for every reader his book. If people are on facebook et al. we need to get there.
