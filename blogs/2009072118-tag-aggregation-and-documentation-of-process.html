tag aggregation and documentation of process

Dan Paluska

July 21, 2009

Please click the above image to see it full size. This is a simple prototype I made to demonstrate a web service I would like. Apologies for the sloppy design but hopefully the point is illustrated. I am interested in documenting process and sharing information. This is all part of my personal process as well. How do all these web services I use act together to create a more coherent story? How can I make it easier for myself? I would love to perfectly organize everything but that takes a lot of time. Maybe there is a way for some things to organize themselves more? I like http://spezify.com for the way it presents a visual version of search results. See http://spezify.com/#/robotics for an example. I like http://delicious.com for the way it autogenerates sublists based on my tags. See http://delicious.com/leinad/robotics I can imagine these types of pages being available to individuals and teams working on projects. In some ways, it's a slightly more automated version of a blog that allows to easier inclusion of information. Sometimes you don't want to write a full blog post or edit a google doc but you'd like something small to be added to the project documentation. The page itself could have a meta RSS feed and would automatically update anytime any of the individual feeds were updated. Friendfeed works this way already but is purely timebased rather than tag, topic, or project based. I would think such a service would be useful for a class or a project team. This is certainly related to list sites as well. See this flickr set as well.
