Android and Iphone Development

pramnab1

February 9, 2009

Daniel Switkin from Google came in today to talk about android application development. Ebay, Shopsavvy made applications which use bar code scanning using smartphones. Android is open source platform (based on Linux kernel) and not a phone but just a platform. It has drivers for camera, bluetooth, flash memory driver, binder(IPC), USB, Keybad, WiFi, and Audio Drivers. Libraries: Surface manager, media framework, SQLite, OpenGL|ES , FreeType, Webkit, SGL , SSL , libc Android Runtime: Core Libraries, Dalvik Virutal Machine Application Framework: Activity Manager, Window Manager, Package Manager, etc. Dynamic Layout of screens using XML where coordinates are given for putting the window on the screen. Two ways to create GUIs: in XML or in Code. GUI code is placed in res/layout, res/values @id/name_for_component gives handles for referencing XML declarations in code. Views: are building blocks. Examples can be: textview, edittext or listview, WebView, MapView, ImageView. Views provide graphic layout, its a rectangle layout. widget adds a specific addition to the function of a scroll, etc. Layouts: controls how views are laid out 1) Frame Layout: 2) Absolute Layout - fixed coordinates. 3) Layouts are customizable (portrait to landscape) using XML res/layout/share.xml (portrait) res/layout-land/share.xml (Landscape) .. .. Basic Components : - Applications 1) Activities - UI Component typically corresponding to one screen.[MOST IMPORTANT]. They can 1) be Faceless 2) be in Floating window 3) return a value 2) Broadcast Receivers - Respond to broadcast Intents (respond to broadcast Intents). Applications can invent and broadcast their own Intents as well. 3) Services : Faceless Tasks that run in the background. (Music Player running in the background). using gps, acceloremeter,etc. 4) ContentProviders: Enable Application to share data. Provides uniform APIs for sharing content. Intents: They are verb, URL , data which express what the user wants. ex. - VIEW, CALL , PLAY. System matches Intent with Activity that can best provide that service. Activities and BroadcastReceivers describe what Intents they can service in their IntentFilters (via AndroidManifest.xml) The system helps in finding the application handler for a specific set of activity. Storage and Persistence ------------------ 1) Preferences 2) Flat File 3) SQLite 4) ContentProvider Only your application can read your data as it's signed but if there is only one key then apps can share the data. Packaging ---------------- 1) .apk files as Android Packages 2) much like .zip file Resources --------------- 1) res/layout - declarative layout files 2) res/drawable - intended for drawing 3) res/anim - bitmaps, animations for transitions .. .. Assets --------- 1) Similar to resources. 2) Read Only 3) Input Stream access to Assets/ Android Application Lifecycle: documentation at android.app.Activity
