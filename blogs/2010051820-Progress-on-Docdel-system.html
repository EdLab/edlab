Progress on Docdel system

Xiaobin Zheng

May 18, 2010

Hui Soo, Ankit and I have been working on the Docdel system this semester. And for now, we've already completed the development of some new functionalities and will start testing it soon. We are looking for feedback on what new functions are needed and how to improve already existing functions. Currently, there are mainly two new functions in Docdel: session management for faculties and embedding of video in personal files. 1. Session management Old Docdel: After logging in and selecting "My classes", you could select existing courses and sessions. However, sessions under each course could not be created/modified under Docdel. To add/delete a session, you need to go to ClassWeb and do it there. New Docdel: Instead of going to ClassWeb to do session management, a faculty member can do it on Docdel now. They can create/delete/edit each session, add reserved items to any session they want and could modify the dates for all sessions together at a time. New sessions would be inserted according to the date and assigned a session number automatically. Sessions whose dates have been changed would be reordered among all sessions according to the dates and those files attached to each session would still be kept with them. Samples could be seen in the attachments. 2. Embedding a video as reserved items Docdel is an E-reserve system where users could upload files and share files within a class. In original Docdel, one could upload PDF/txt files by specifying the URL where the file is located(For example, from the Teachers College library). With the suggestion of Brian, we added the function of embedding a video using an embed code. So now, we can go to YouTube and copy the embed code of a video, paste them when we want to add it, and the video will show up in the reserved item list. This embedded video could be assigned to a certain session or to a certain personal folder in the same way as other type of files do. 3. Underlying database changes Old Docdel relies on the data coming from databases in ClassWeb such as course information, session information, and registration information, among others. Since we want to make Docdel a self-contained system, we don't want to fetch data from databases in ClassWeb. We designed relational tables in Docdel to model those information and insert data using feed files coming directly from Teachers College so that Docdel would not rely on ClassWeb any more, either functionalities or underlying databases. And we would also update information directly using feed files in future. Above is the main progress on the Docdel system. We will present it soon and collecting feedback accordingly but in the meantime we welcome any idea about how to further improve Docdel. Thanks.
