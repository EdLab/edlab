Understanding the New Discovery Landscape: Federated Search, Web-scale Discovery, Next-Generation Catalog and the rest

pramnab1

July 13, 2010

Background: Hui Soo, Shaoqing, Jo Agnitti and I have been working on building and testing out an open source Integrated Library Management system called Koha. Where we are: We have currently tested out a successful deployment of Koha and are in the process of importing data from our current Integrated Library System. You can evaluate the Koha system by logging into our test setup (only accessible inside TC network and requires one to be hardwired in or on VPN). For the username and password please get in touch with Joann or Shaoqing. Research: We have been consistently monitoring the library search space with respect to the developments done in the library cataloging and how to make the patron experience much more enriching. In regard to this there was a recent webinar done by Serial Solutions on the topic of Understanding the New Discovery Landscape: Federated Search, Web-scale Discovery, Next-Generation Catalog and the rest. I am mentioning the key points for everybody to get a good sense of where the new developments are heading and would like to sought out any feedback of what ways we should proceed our current endeavors to include that in the Koha ILS which we are currently playing around this. Please shoot an email to either of us or include a blog post. So not boring you enough here are the key details which I found out: Today a Modernized Interface for a library catalog seems to have all of these functionalities which are in tandem with the features offered by Google, Amazon, etc:

  1. Single Search box
  2. Query Tools (Did you mean, type-ahead)
  3. Relevance Ranked results
  4. Enhanced visual displays
  5. Recommendation services
A good example of a library having a good User interface design is Mississippi Library. http://www.olemiss.edu/depts/general_library/ To get a background of the library cataloging and how it evolved please refer to the wiki page of OPAC. New Library Discovery layer and how it interfaces with other systems to make the search more comprehensible and scalable to not only our library but also other libraries The latest new developments happening inside the colleges and the libraries:
  • Teaching platform replaced by Moodle
  • Replacement of ILMS- emphasis on front end - Aquabrowser(tag clouds)
  • New building - integrating experiential lerning plus other services
  • Accountability - ROI, statistics
  • Research - repositories
  • RFID
  • Improving access
  • E-only catalogue
There is also a new offering offered by serial solutions which is meant to assist libraries in getting the content in a more authoritative fashion to users who are right now relying on search engines for ease of use. Background about Summon: Summon a new library discovery search with much more information provided by it as to the likes of, however not keeping the library at the backstage but at the front. Salient Features of Summon:
  • It mines all the full text and has abstract over the mouse over.
  • It has partners like LexisNexis, Proquest, etc.
  • Find by subject terms, content types, narrow by dates, etc.
  • It also has a mobile platform presence (iPhone)
An instance where you can evaluate Summon deployed by University of Huddersfield here. Summon seems to be an intuitive enough to even search by citation type for journals. More information available on Summon is available here. You can view the Seminar Webcast for the event here and also see the presentation slides which are attached with this blog entry. We would be more interested in hearing from you all (including Gary) as to what features you think would be useful for the next generation ILS for TC.
