JavaScript Namespacing

Connie Liu

January 21, 2011

One of the things that I'm working on is JavaScript namespacing. Here are my finds thus far, which I hope some of you may find useful: First, namespacing is a programming technique that helps protect your variables from other programs by helping to avoid conflicts or data changes that you would not want to occur. A namespace is a space or a region where a certain name (variable) is valid. An example to make things clearer is: Let's say there are two people in a company with the same name, John. In order to differentiate between the two, you would perhaps call one “John from IT” and the other “John from HR.” This avoids any confusion and you know that if you were to need IT help, you would call “John from IT” as opposed to if you were to call just “John” with your request possibly falling on the guy from HR, which is not helpful to say the least. For JavaScript, please note that there are two scopes, global (file level) and local (function level). And in terms of writing code, like JavaScript, as well as any other programming languages, it's a good idea to use namespacing to make code more presentable, to make future development of code easier, and to provide a structure to make something more scalable.
