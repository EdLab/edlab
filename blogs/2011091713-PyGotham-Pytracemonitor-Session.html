PyGotham Pytracemonitor Session

Yudan Li

September 17, 2011

Purpose * Record all objects in an executing program * Outputs from print statements and terminating exception can also be recorded * Stop execution on conditions, such as too many lines or too much time * Stored in file or print on screen * read file after execution * Stored output from print statements * Store full traceback from terminating exception common setup * Line number * Name of current function * local variables Limitations * sys.settrace(trace_function), trace) function is executed for each event *events- line, call, return, exception, and others * sys.settrace is read only and cannot be stopped except by exception * threads not tried, but will probably work additional uses * new code can be inserted ahead if certain value is found * execution profiler, recording time that each line is executed * perform tests similar to unittest * Memory utilization at each line * stop if program is in infinite loop * Report condition of execution program, to ensure it is still working program structure * the system is a skeleton that can be added to * independent of python version or system utilities * parameters read in from configuration file * started with traceon, stopped with traceoff * can be run interactively in raw command line Python
