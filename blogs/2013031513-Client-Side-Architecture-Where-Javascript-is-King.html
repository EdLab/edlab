Client Side Architecture, Where Javascript is King

Sharath Gururaj

March 15, 2013

***Private Please Given that we've decided to be systematic about our client side, (i.e., we'll have to use the whole shebang--framework, build tools, testing tools) to the point that perhaps the django-python layer will be very thin, it is worthwhile to explore the javascript -- the language itself (without getting bogged down by the DOM, browser etc) 1. Javascript is a functional language. Functions are first class objects, they can be tossed around like any lowly int 2. Everything is an object, including primitives like int and even functions. It is possible to define member variables for functions! 3. But Javascript is not object oriented. It has zero support for classes, inheritence, polymorphism, type inference etc. 4. Which means that an object is just a hash of key/value pairs. keys are strings, values are anything: fucntions, objects, primitives etc 5. Javascript is single threaded. At first sight, this is at odds with the fact that a lot of Javascript code is asynchronous, and thus event driven. Worth getting to know better! 6. Its quit fast. On the server side, with Node.js for javascript vs HotSpot for java, the speeds are comparable, with javascript around half as slow as java On the browser, my estimate is that the V8 is almost as fast as python. 7. Crazy things are possible. Two of my favorite is: toString() on a function gives you the function definition as a string! eval() lets you interpret a string as javascript code at runtime! NOT for the faint hearted! A good reference for javascript is the book Javascript for Ninjas, written by the author of juery, John Resig available in the edlab office. Given the power and flexibility, it is exceedingly clear that without proper testing, writing/maintaining any non trivial code is going to be futile. A compiler cannot replace test cases, but a good set of test cases can make up for a lack of compiler. There is also another flavor of java called coffeescript, which compiles to javascript and is much more simpler. What are your thoughts and opinions about javascript? How would you rate your experience with it?
