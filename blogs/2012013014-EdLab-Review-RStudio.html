EdLab Review: RStudio

zhou

January 30, 2012

RStudio is an open source integrated development environment (IDE) for the R statistics software. If you don't know about R, R is an open source programming language and software environment for statistical computing and graphics. R is widely used by researchers and enterprises in statistical analysis and data mining. I had been playing with R in the past two weeks in a data analysis project for Vialogues. As a long-time user (>5 years) of SPSS, I found R highly powerful and flexible being a open-source software. However, I also felt frustrating in learning R because it would take some time to grasp all the functions and syntaxes. In SPSS you can simply import the data, click a few menu items, set the parameters, and click “run” to do a t-test or regression. But in R, you have to do ten times more things. Few established statistical procedures are built into R. You have to write pretty complex scripts even for tasks as simple as importing the data and creating bar graphs. And it's especially frustrating in trying to figure out what functions to use to do seemingly simple tasks and guessing what are wrong with your scripts if your desired outcome is not seen. It is said the best thing about R is it was designed and created by statisticians who made R extremely professional. But the worst thing about R is also it was created by statisticians who made R too hardcore. I'm feeling exactly the same. So you see how glad I was to discover RStudio today because it provides a neat graphic user interface to use R more easily. Pros: I downloaded and installed RStudio in 5 minutes and created a new project based on my existing R project. RStudio picked up my existing data with no problem and executed my scripts producing the same results as using R alone. RStudio makes some operations much more user-friendly than using R alone. It centralizes many windows involved in using R including the main R console, file editor/data viewer, workspace/history monitor, and file/plots/package navigator. So I no longer have to switch between many applications such as R, text editor, Finder, Terminal, etc. RStudio provides instant code hint in writing R scripts. And I find it very handy to monitor and change project variables in RStudio which otherwise have to be done by writing scripts. RStudio does more than providing a GUI for single users to use R. It also provides the option to install on a web server so that the researchers from the same institution/corporate can reach the same customized, powerful data processing environment. What's more, version control is also built in RStudio so that it supports larger projects involving multiple researchers. I saw some people have installed RStudio with Amazon EC2 service and here's a great tutorial. Cons: As a new project, RStudio still hasn't made data processing jobs as easy as using SPSS. I feel lots of new features can be added to RStudio such as: - Importing data from Excel/CSV/XML files in addition to TXT files (TXT is the data format officially supported by R) - Exporting variable values into external files - Interactive helper for executing typical statistical procedures such as t-tests, regression, etc. In R/RStudio typically you have to write lines of codes in order to run analyses such as: > anova( object, ... ) (run ANOVA on one or more data objects) or > lm(formula, data, subset, weights, na.action, method = "qr", model = TRUE, x = FALSE, y = FALSE, qr = TRUE, singular.ok = TRUE, contrasts = NULL, offset, ...) (run linear model) If you don't know the function name, syntax, or parameters you have to spend the time to search and understand the R documentations, which is quite annoying. So it is preferable to allow the user to navigate to or search for a task, compute required params step by step, and run the operation without writing codes. At this point of time such interactive helper is not implemented yet in RStudio. To summarize, RStudio does a nice job in interpreting the basic R operations with a user-friendly GUI. It would be most valuable by enabling typical statistical operations in its features. These new features can be optional by installing additional add-ons based on user need. And it's ideal to allow user to add their own custom operations in addition to a base of operations. EdLab Relevance: The Data Dashboard project! We can install RStudio Server on an EdLab server and develop packages for R to run customized analysis for different EdLab products. Then on the Data Dashboard the project admins can conduct all kinds of analysis and produce nice tables and graphs. For people who have difficulty in finding free SPSS licenses, R could also be a nice alternative to use. Our Takeaway: RStudio can make lots of researchers (especially students and corporations) to switch from the intimidatingly priced stats software such as MatLab, SPSS, and Stata to the open source R. Both R and RStudio are free. So why not give them a try?
