ReCaptcha: Digitizing Books One Word at a Time

Will Heublein

August 13, 2007

In a similar vein to Blackle, the reduced-power search screen which Samantha posted about a few weeks ago, a project based at Carnegie Mellon called 'ReCaptcha' takes a routine web process and ties it in with a broader goal. Here, the routine process is user authentication or verification using a CAPTCHA (Completely Automated Public Turing test to tell Computers and Humans Apart), and the broader goal is the correction of ambiguous OCR (optical character recognition)-scanned text in the public domain. Captchas typically present a user with a visually distorted string of letters in a web-based image, and ask the user to enter the same string in a plain text form. This simple test deters spam postings on forums and user generated sites because automated spam-posting bots are unable to read Captchas as well as humans. Instead of a randomized string, ReCaptcha makes use of two images of different words, and two forms for text entry. One of these words is known to the server, the other has been flagged as ambiguous by OCR software. If the users enters the known word correctly, it is assumed that they were also correct about the other previously undetected word. Nevertheless, several users must verify the ambiguous word before it is considered solved. The ReCaptcha team estimates that more than 150,000 hours of work each day are spent on Captchas, but does not yet provide statistics on how widely used their tool is, or how much OCR text has been verified through their system.
