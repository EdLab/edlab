WYSIWYG Editor for EdLab Website

zhou

May 19, 2015

--- keep private --- As I mentioned to Dr. Brian Hughes we are going to upgrade the EdLab website from the broken Drupal 4.6.4 to 6.35 tomorrow (yes, tomorrow May 20, 2015), he asked if we will have a What You See Is What You Get (WYSIWYG) editor. I think it's a great idea but I don't have time to research the best WYSIWYG for our needs and install/test it before tomorrow. How disappointing. However, over my lunch today I did look around for the few options we could have which we can use at a later time. And I'd like to know what people at the EdLab like.

Candidate 1: Markdown

This is my personal favorite. It indeed will take a few days for people to get good at. But once you do I bet you will never miss plain or rich HTML editors. Basically Markdown allows one to write clean, structured HTML with simple syntax in plain text. It's been there for a while, most popular among programmers and professional web writer, but increasingly popular these days. The EdLab D&R has been using BitBucket for years which uses Markdown. Some exemplary markdowns are: 
# Heading 1
## Heading 2
### Heading 3
*italic*
**bold**
***bold-italic***
[edlab](http://edlab.tc.columbia.edu)
![edlab logo](http://edlab.tc.columbia.edu/
files/images/edlab_logo.png)
These will be converted to HTML: 
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Header 3</h3>
<i>italic</i>
<b>bold</b>
<i><b>bold-italic</b></i>
<a href="http://edlab.tc.columbia.edu">edlab</a>
<img src="http://edlab.tc.columbia.edu/
files/images/edlab_logo.png" alt="edlab logo" />
And for people who really love clicking buttons, yes you still have the buttons (but only those 99% people will ever need)! Note the sign in the screenshot below. It will display a live preview of your markdowns! For a complete guide into Markown, read this article.

Candidate 2: Rich HTML Editors

These include the the traditional WYSIWYG editors including TinyMCE, CKEditor, NicEdit, and so on. There is no learning curve for rich HTML editors because it's just how one writes Word and Google Docs. The reasons why I vote down this option are: 1) My writing flow is frequently disturbed because I have to leave the keyboard and click the mouse. 2) If the HTML I write is ill-formatted it may break the whole page. 3) The more features, the buggier.

Candidate 3: Keep Using the Plain Text Editor

-- Some people may prefer continuing to use the current plain text editor. If you think this way please also let me know. So, what do you like?
