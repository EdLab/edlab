User info reporting standards

Josh Lopez

August 7, 2012

Keep private please. Data Dashboard needs a way to keep track of who is viewing a particular site. The plan is to use CAS usernames to uniquely identify users, but there's no standard method of providing this information. Right now each app does it a little differently: Vialogues - javascript object Ednode & Pundit - included in the href string of an <a> element Research Broker - not available With the impending launch of Survey Sidekick and New Learning Times, and the need to add Data Dashboard tracking to all of these apps, it might be a good idea to have every site use the same method for making this, and other basic information, accessible. The alternative is to require every app to report the current username to Data Dashboard via a provided function. This isn't necessarily a problem, in fact I've already written the necessary function, but I'd like to get everyone else's opinion on this issue. Should every app make the username accessible in the same way, so Data Dashboard can access it automatically, or should every app be responsible for reporting the username to Data Dashboard manually?
