PK 2.0 Design

bhughes

February 27, 2008

Hi everybody, You've been waiting a long time for this, so I'll cut to the chase: Dan and I have cleared the initial project planning stage of refreshing PocketKnowledge, and we are looking for help. Yes, that's right, PK 2.0 is in progress. Here are some details:

  • Recode existing application to fix bugs
  • Use freely available Yahoo! Omnifind search tool to improve search
  • Update UI with changes to reflect user feedback and new functionality
  • Intersection functionality remains essentially the same, but UI will be more modular
  • UI will not be optimized for viewing image collections (so sad)
  • Meet robust accessibility standards, but also include Ajax functionality
  • Possible community tagging
  • Possible administrator “locked” pockets
  • No private/shared pockets, only private/shared items
And there are a bunch of other tweaks and changes too, and the challenge will be to keep track of them all and to elegantly integrate them into the application. Does this sound like your kind of project? For the next four weeks we will be refining the UI (possibly employing paper mock-ups & informal focus groups) and defining data tool requirements (standard reporting, etc.). If you are interested in lending a hand or spear-heading one of these tasks, please let me know. Also, I know Dan is eager to talk through these changes with folks, to iron out the details. Stop by and see how his work is going — he is already at work coding the system core. Our plan is to roll out the refreshed PK (what we affectionately like to call the "Research Edition") for the fall semester. Let us know if you have any other ideas to make development or the roll-out a success. Brian
