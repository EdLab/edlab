Javascript resource

fanwar

March 14, 2010

Several projects at the EdLab are trying to do some serious work with javascript, a language that is probably 'hacked' at the most (I'm guilty of that :)). Designing and planning your code with it is usually an afterthought and this can cause some problems when you get more complex javascript-driven UIs. I found this site as a good resource for javascript in case people are interested (especially those building front-facing applications): I especially recommend the "Private Members in Javascript" section which talks about scoping issues.
