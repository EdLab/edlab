Pagination In Django

Yudan Li

April 11, 2011

Recently, I've been working on a few features of the Events Calendar project. One interesting feature is the pagination effect on the event aggregator's page. Pagination can be just considered as the process of dividing information (content) into discrete pages. The Django framework provides the helper model Paginator which can be applied to other user-defined model. Once instantiated, the paginator allows the extraction of a certain page. An extracted page object then contains the set of data that falls on the current page. The page object also contains the information including (but not limited to) the list of all the existing page numbers and whether there's a next/previous page. The advantage of utilizing the Paginator model is its efficiency and the clear but powerful interface it provides. For the front end to get a page of data, two parameters are sent out: 1) number of items per page and 2) the page number that is currently needed.
