Testing edlab applications with BrowserStack

Sharath Gururaj

April 10, 2013

private please Hi All, Today, I would like to discuss BrowserStack, which may prove to be invaluable for future cross browser testing. Currently at edlab, the only way to test on IE is to fire up an old dell windows laptop and bring up ie, or install a virtual machine and run IE on it. But if you think having a spare laptop handy or installing an entire (virtual) operating system to get hold of a browser, you're not alone! It is ironic indeed that browsers let you access to the worlds information and tools, but getting access to a browser itself should be so roundabout. Whats the solution? access a browser in a browser, of course! :) BrowserStack does exactly this. You connect to a virtual machine over a cloud, and access the exact browser version you want on the operating system you specify, all from within the browser. The cool thing is that there is a way to access your local files from the remote machine and open it from a browser. It also lets you access a locally running server. At EdLab, we're trying out Karma for day to day development. It has the ability to connect to a remote browser and run tests there. But it might be an interesting project to hookup Karma and BrowserStack. Any volunteers? :)
