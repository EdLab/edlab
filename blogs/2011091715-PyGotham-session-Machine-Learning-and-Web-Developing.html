PyGotham session: Machine Learning and Web Developing

Yudan Li

September 17, 2011

Machine learning package Download and install package: * libyaml (on Ubuntu you need libyaml and libyaml-dev) *Download the code: * git clone https://www.github.com/thatdatabaseguy/ml4webdevs.git * cd ml4webdevs/setup * ./bootstrap.sh Why you should care * Ever wanted to give your users meaningful recommendations? * How about finding similar documents? * detecting spam/fraudulent transactions? What machine learning is NOT? * re.compile('<\s*h1[^>]...) * ETL/ data cleansing * twitter_id2facebook_id OMG I <3 #mashup! * Big Data or using a NoS?QL data store * Full-text search *Reporting and/or data visualization This IS rocket science (sort of) * We will use convenient abstractions and vanilla algorithms for today's class * BUT, more advanced machine learning algorithms require strong understanding of linear algebra Some bubble-bursting * the following does not exist:   * import machine_learning   * learn(my_data)   * predict(my_new_data) What does exist * Lots of cool open-source libraries   * nltk (highly recommended for NLP as well)   * scikits.learn (fast, uses scipy)   * pybrain   * pattern   * pyml Supervised learning * have some known, correctly labeled data * extract some features from the data, forming a set of training tuples * use an algorithm (there are several) to learn from the data and produce a model * the model can be used in the prediction of labels from unseen instances Feature extraction * this is the most important part you need to think about in supervised learning models * denormalize object graph or relations into a tuple of binary attributes or real numbers* single features can be complex(contains synonym for "jell: after synonym for "peanut butter') * stored in a dictionary or a vector Regression * predicted value is continuous * very applicable in the financial sector (eg predicting risk of an asset) * ALSO very applicable to system s administration (how many users should I expect on-sites? Given the load on my databases, how much latency should this be?) Classification * predicted value is discrete * discrete value can mean:   * binary values (eg spam/not spam)   * most likely category   * probability distribution over categories example case: Classifying political tweets * two top political hashtags on twitter:   * #o2 (progressives 2.0)   * #tcot (Top Conservatives on Twitter) * Train model with tweets for these hashtags * Classify new tweets as learning material What about intersection of A and B? * indeed, intersection is ambiguous * user set complements (A-B and B-A) Old Man Bayes' Law * Promised to stay light on the notation but a naive bayes classifier * hellow world of supervies learning * single classification (maximum likelihood) * "naive" - assumes features are independent * very efficient to train (multiply probabilities) commands from classifier import PartisantweetClassifier classifier = PartisanTweetClassifier() classifier.fetch_twitter() classifier.train_model() classifier.accuracy() This is fine in the shell, but I promised to talk about web applications saving the model * Two methods that worked for me: * stuff it in the cache (for small models) and keep a copy on disk * for larger models, just put it on disk on the datamine box and write a simple RESTful service for classification
