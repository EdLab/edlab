How Recommendation System Works

Pengfei Li

May 31, 2011

There are two major methods in designing a recommendation system: content-based method and collaborative filtering method. I would like to introduce some basic mechanism of the algorithms and introduce how these methods are been used in commercial products. First, let's talk about content-based method. There is a nice paper, "Content-Based Recommendation Systems," which talks about different content-based algorithms. It is a nice overview and gives the basic idea of each algorithms and their usage in practice. Content-based method means that we get features from the content of item, in many cases they are words which describe the item. For commercial products, Pandora maybe using content-based method, but amazingly all features are scored by human beings, see how Pandora works here; IMDb maybe also using content-based method because it is full of content of movies. The other method I want to talk about is collaborative filtering method. The main idea of this method is to find a group of people who share the same interest with you. These people could be determined by the similar ratings on items. Let's call these people the neighborhood of the current user. If an item is liked by most people in this neighborhood, then it is very possible to be liked by the current user. This is collaborative filtering method. Netflix is using collaborative filtering method, see how Netflix work here. I guess if a recommendation system involves rating on items then it is probably using collaborative filtering method. There is another kind of collaborative filtering method called item-to-item collaborative filtering method. It was developed by Amazon. As we know there is a huge inventory of items on Amazon, so the traditional recommendation algorithm will not work well because of time complexities. As such, Amazon developed its own algorithm called item-to-item collaborative filtering. The paper describing this algorithm is "Amazon.com Recommendations Item-to-Item Collaborative Filtering" published on IEEE, it's not free online, but you can get it from Columbia University's Library. The main idea of this algorithm is that, instead of finding neighborhood users who share the same interest on items with the active user, it finds the items been purchased together with the current item. Among these items, you can find the similar ones by cosine similarity and calculate the statistics on what percentage of people buy each one. To my understanding, this approach is incrementally maintainable, which makes it easy to rebuild index in the offline part. The online part need less time than the original collaborative filtering method because the system do not need to compare the active user with all users in the database to find the similar neighborhood. I hope this could give you some idea of how recommendation system works. There are also some other kinds of recommendation systems.
