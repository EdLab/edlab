Andorid and Iphone Development continued...

pramnab1

February 9, 2009

Emulator : QEMU - based ARM emulator Eclipse - preferred development environment, needs an android development plugin. android source code DDMS - trace allocations, check heat, etc. (used for debugging android applications)

Hello world Application

- 1) Install plugin in eclipse 2) Specify Android sdk location 3) create a new android project using eclipse 4) R.java - Resources file (don't edit it) 5) create a new java class calling it HelloActivity.java and have an onCreate method in which you call the function of printing "hello world" 6) hello_activity.xml - in res/layout folder hello_activity.xml (for layout) AndroidManifest.xml controls everything for the application and it also has the package name much like the info.plist for iPhone SDK, Intents and manifests are specified here. 7) Run as an android application and then set the Target to the QEMU or if the android device is plugged in use it for testing the application. zxing - an application for barcode reader. Applications which can be build for android: -------------------------------------
  • 3d games using OpenGL
  • blogging application
  • accelerometer apps
  • compass apps
  • voice over ip clients using wifi
currently all the third party applications are in Java. Differences between Android and Iphone 1) Fix a bug 2) Lower level access to camera as compared to iPhone- can't do preview screens 3) Anything can be uploaded on to the android application market not as rigid as Apple App Store.
