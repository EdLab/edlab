Making our projects at EdLab more secure

Weiqi Ma

May 16, 2010

I have been taking a course about software and network security this semester and really interested in security issues regarding projects at EdLab. So there are some questions we might want to ask ourselves: 1. Is there any misconfigured web server at EdLab? As far as I know, there are actually a lot of web servers running at EdLab, ranging from Apache Server to the Microsoft IIS server. Are we sure all those web servers are properly configured without any kind of potential security vulnerabilities? If not, it would raise a few potential security problems, since misconfiguration of web servers could result in huge security vulnerabilities. 2. Is there any security problem with the network setup at EdLab? The servers hosting sensitive data or projects at EdLab are only accessible from inside the Teachers College Network. However, it does not guarantee that attackers from outside could not get what they want. In fact, it is pretty easy to log on a host that is supposed to be inaccessible from the outside network using the SSH port forwarding service. 3. Is the password well protected by each user at EdLab? Password management could be a big problem sometimes. I don't know about others but I often tend to choose my password as something easy to remember for me, which probably means it is easy to guess or brute forced by attackers. Therefore, picking an appropriate password is very important to protecting both your own account and sensitive data at EdLab. And it's better not to write down your password somewhere in case of password being stolen by others. 4. Are our projects well designed and implemented considering the security issues? In order to develop a secure software product, we could build a threat model to identify and mitigate all potential threats in our projects. A threat model is a description of a set of security aspects, by which one can define a set of attacks to consider. It is the core element of secure software development process, in that it determines potential security design issues, drives the security analysis and testing process. And above all, the best thing about threat modeling is that you can identify and mitigate potential security issues at the early stage of the development process. Hence, the security problems can be fixed at a relatively low cost. Threat modeling is easy to do using Microsoft Threat Analysis & Modeling tool, which provides general guideline and comprehensive automatic evaluation tools to simplify the threat modeling process. And amazingly, you don't have to be a security expert to use this tool. You don't even have to know much about programming. All you need to know are the business requirements and application architecture of your project. Then, you can leave the rest to this fantastic tool to produce a feature-rich threat model.
