Data Dashboard Update

Josh Lopez

June 15, 2012

Keep private please. Data Dashboard's front end tracking functionality is ready. When we add it to our sites, we can keep track of exactly how users are using our apps in far more detail than even Google Analytics can tell us. For example, Data Dashboard will tell us how users are choosing the videos they're using for new Vialogues, how much time passes between postings and responses on Research Broker, and exactly when a user decides to sign in on any site. With all the information we're going to be collecting, the question that comes up is how to display it all. If we're just one or two dimensions we could just look at boring line graphs and pie charts, but this isn't 1801 so maybe we can do something at little more interesting. We're also going to need some ideas for displaying data in more dimensions. I think this falls right in line with Janice's post about Processing. Think about what kinds of reports we might want for our apps and check out some of these examples: http://thejit.org/static/v20/Jit/Examples/RGraph/example1.html http://trends.truliablog.com/vis/metro-movers/ http://mbostock.github.com/d3/ex/bubble.html http://mbostock.github.com/d3/talk/20111018/calendar.html
