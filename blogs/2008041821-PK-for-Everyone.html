PK for Everyone

Dan Mallinger

April 18, 2008

I just found a great site Darwin Online, which touts 90,000+ pages of manuscripts, sketches, and field notes from that guy with that funny idea. It's definitely worth a look, but something else occurred to me... When looking at the site, I still wanted the PK like suggestion tools. I wanted to know what was popular, what others like me looked at, etc. I think it's possible to build a generic tool to do something like this. The key is to have javascript (js) widgets that you can drop into a page (like Google Analytics). Then, the js hits our server and passes the url and the unique cookie of the user viewing the page. This would allow our servers to maintain usage data WITHIN sites to power suggestion tools. This could be done to analyze many sites at once, while providing links only within sites. (This way, one company wouldn't have to promote another). A second js widget queries our servers when the page renders. It tells our server that a user with cookie id X is viewing url Y from company Z. Our server then returns a list of "suggested links" from Z, which display nicely one the page, alongside the other content. What this means, is that Darwin Online (a big archive) could suggest meaningful content to users based of usage patterns without having to build a system. All they would have to do is copy and paste some javascript into their pages (again, just like everyone does for Google Analytics). This would give the power of PK to any other archive! It's an interesting idea I think. Instead of pushing everyone to use our archival system because it offers the best user experience, we offer the same user experience to other archives. If the widgets were powerful, we could offer a very interesting service. And maybe that's the real future of libraries, not storing content, but providing the best means of accessing it.
